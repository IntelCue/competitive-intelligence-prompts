# Topical Authority Map

## Goal
Map the topic clusters and internal-link structure you'd need to build genuine topical authority in a niche — the pillar-and-cluster architecture that signals depth to search engines and AI answer engines alike.

## Best for
Planning a content architecture from scratch, or restructuring a sprawling blog into something that ranks and gets cited.

## Prompt
```text
You are an SEO strategist designing a topical authority map for {your_company} to become the definitive resource on {core_topic} for {audience}.

Build the architecture:

1. PILLAR TOPICS: The 4-7 pillar pages that anchor the niche — broad, high-value hubs.
2. CLUSTER CONTENT: For each pillar, 6-12 supporting articles (the cluster) that cover subtopics, questions, and long-tail intent. Give working titles and the search intent each serves.
3. INTERNAL LINKING: How clusters should link to pillars and to each other to concentrate authority.
4. ENTITY COVERAGE: The key entities, concepts, and questions that must be covered to be considered comprehensive on this topic (think: what an expert would expect to see).
5. SEQUENCING: A build order — which pillar + cluster to create first for the fastest authority and traffic gains.
6. AEO ANGLE: For each pillar, the specific questions an AI assistant (ChatGPT, Perplexity, AI Overviews) is likely to ask, so the content is citable, not just rankable.

Rules: Make titles specific and intent-driven. Cover the topic the way a domain expert would, including the unglamorous subtopics. Note where keyword validation is still needed.

Core topic: {core_topic}
Audience: {audience}
Our site (optional): {your_url}
```

## Output format
- **Pillar list** — the anchor hubs and what each owns.
- **Cluster map** — per pillar, the supporting articles with intent.
- **Internal-linking plan** — how authority flows through the structure.
- **Entity & question checklist** — what "comprehensive" requires.
- **Build sequence** — what to create first and why.
- **AEO questions** — the answer-engine queries each pillar should win.

## Optional inputs
- Your existing content, so the map builds on what you have.
- Competitor hubs to benchmark depth against (see [content gap](content-gap.md)).
- The specific AI engines you most want to be cited by.

## Notes
Topical authority is a multi-quarter play, not a single post — the sequencing output is what makes it executable. The AEO angle is increasingly the point: AI answer engines reward comprehensive, well-structured topic coverage, so design for citation, not just for the blue link. Validate the cluster keywords with a real tool before committing the calendar.
